## 1. Goal (目标)

对当前 `multibeam` 项目进行一次基于代码仓库现状的结构分析，帮助用户快速理解：

1. 顶层目录与职责划分
2. 核心运行入口与主流程
3. 模型、数据、工具层、业务层之间的关系
4. 输出物与历史治理文档的位置
5. 当前结构上的优点、耦合点与后续可关注方向

## 3. Constraints (约束条件)

- 本次任务为**只读分析**，不进行功能修改或重构
- 结构判断必须以仓库当前内容为准，而不是仅复述 README
- 若 README 与实际结构存在偏差，应以实际文件为准并明确指出
- 不臆测未读取文件中的实现细节

## 5. Product Acceptance Criteria (产品验收标准)

- 明确哪些部分是运行时依赖，哪些部分是离线资产或历史产物
