## 1. Goal

将 `multibeam` 项目升级为“综合最优”算法版本：优先降低漏测率，其次控制重叠和总测线长度，在不推翻现有主测线/扩展骨架的前提下，引入全局评分与补洞机制，使输出结果更稳定、指标更可信、答辩说服力更强。

## 2. Deliverable

### 2.1 代码交付物

| 文件 | 变更类型 | 说明 |
|---|---|---|
| `multibeam/Planner.py` | 重构 | 引入候选线评分、唯一覆盖统计、补洞 pass |
| `multibeam/models.py` | 修改 | 补充评分/指标相关数据结构 |
| `multibeam/planner_utils.py` | 修改 | 新增评分、网格覆盖、候选线评估工具 |
| `multibeam/Coverage.py` | 复用/适配 | 复用网格坐标与覆盖口径 |
| `main.py` | 修改 | 支持新算法参数入口 |
| `README.md` | 修改 | 说明综合目标算法与指标口径 |
| `tests/` | 新增 | 增量覆盖、评分、补洞、指标约束测试 |

### 2.2 文档交付物

- 算法优化执行计划
- 指标口径说明
- 验证结果与清理回执

## 3. Constraints

### 3.1 功能约束
- 保留 `main.py` 作为默认入口
- 保留现有输出目录结构，允许在 `metrics.xlsx` 中新增统计项
- 不要求完全重写测线生成器，不引入全新求解器

### 3.2 算法约束
- 用户目标为“综合最优”：先压漏测，再控制重叠与总长度
- 不能再用 `总覆盖面积 * (1 - n)` 充当全局真实覆盖面积
- 真实覆盖指标必须基于唯一覆盖口径计算

### 3.3 工程约束
- Python >= 3.12
- 优先复用现有 NumPy/Pandas/Shapely 依赖
- 新增复杂度应保持在本地可运行范围内，不引入外部服务

## 4. Acceptance Criteria

### 4.1 指标可信度
- [ ] 全局统计输出“总扫测面积 / 唯一覆盖面积 / 重复覆盖面积 / 待测海域面积”
- [ ] `覆盖率` 基于唯一覆盖面积计算
- [ ] `漏测率` 始终位于 `[0, 1]`
- [ ] `唯一覆盖面积 <= 待测海域总面积`

### 4.2 综合评分机制
- [ ] 候选测线评估至少包含：新增唯一覆盖收益、重复覆盖惩罚、长度惩罚、弯折惩罚
- [ ] 每轮扩展按评分选择或拒绝候选线，而不是无条件接受
- [ ] 评分参数具备明确默认值与注释

### 4.3 补洞机制
- [ ] 在主规划完成后执行一次补洞 pass
- [ ] 补洞 pass 仅针对低覆盖/未覆盖区域
- [ ] 补洞后漏测率不高于补洞前

### 4.4 验证与稳定性
- [ ] 新增自动化测试覆盖评分与指标边界
- [ ] 运行 `python main.py` 成功生成输出
- [ ] 输出 Excel 中不存在负漏测率

## 5. Non-goals

- 不改训练数据与随机森林模型
- 不引入整数规划、遗传算法等完全新求解框架
- 不以“最短总长度”作为唯一优化目标

## 6. Autonomy Mode

- **Mode**: `interactive_governed`
- **Internal Grade**: `XL`
- **理由**: 涉及指标口径、候选评分、补洞策略、测试与文档多个子问题，需要分 wave 执行与验证

## 7. Inferred Assumptions

1. 用户接受对输出指标口径做纠偏，即便与旧版数值不一致
2. 用户更看重结果说服力，而非保守保持原算法行为
3. 用户希望在当前分支上直接推进实现，而不是只写方案

## 8. Verification Plan

1. 单元测试：候选线评分、唯一覆盖面积累计、漏测率边界
2. 集成运行：`python main.py`
3. 结果检查：Excel 指标范围合法，且新增字段存在

## 9. Rollback Rule

若新评分/补洞机制导致主流程无法完成，则保留旧主测线生成骨架，仅回滚评分决策与补洞注入层，不回滚已修正的指标口径。
