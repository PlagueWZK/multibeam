# 双网格输出结果核查需求文档

**文档ID**: REQ-2026-04-10-002  
**创建时间**: 2026-04-10  
**状态**: Frozen

---

## 1. Goal (目标)

核查用户昨晚已运行出的双网格版本输出结果，重点确认：

1. 新增细网格统计指标是否已落地到输出结果
2. 新口径下覆盖率/漏测率是否落在合理范围内
3. 新旧口径是否表现出预期差异（旧口径可能高估，新口径应更保守且不为负）
4. 对“在侧线规划图上渲染细网格状态”这一后续建议给出评估与下一步建议

---

## 2. Deliverable (交付物)

- 一份中文核查结论，说明：
  - 检查了哪个输出目录
  - metrics 中的关键结果是否合理
  - 是否发现异常或需进一步验证的点
  - 对细网格状态可视化建议的判断

---

## 3. Constraints (约束条件)

- 本次优先做结果核查，不直接修改业务代码
- 结论必须基于实际输出文件，而非只看代码
- 若仅能确认“指标范围合理”，必须明确区分于“算法正确性已完全证明”

---

## 4. Acceptance Criteria (验收标准)

- [x] 定位最新输出目录
- [x] 读取 metrics 结果
- [x] 检查新口径覆盖率/漏测率范围
- [x] 对新旧口径差异做解释
- [x] 评估可视化建议

---

## 5. Delivery Truth Contract (交付真实性约定)

- 基于实际输出文件和静态代码交叉核查
- 不声称已经完成新的图层可视化实现
- 不将一次输出核查等同于全面回归验证

---

## 6. Non-goals (非目标)

- 不在本阶段直接实现图中细网格渲染
- 不在本阶段重新跑完整大规模规划
