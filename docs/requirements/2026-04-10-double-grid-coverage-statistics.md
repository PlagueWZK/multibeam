# 双网格覆盖统计与规划状态需求文档

**文档ID**: REQ-2026-04-10-001  
**创建时间**: 2026-04-10  
**状态**: Frozen

---

## 1. Goal (目标)

为 `multibeam` 项目设计一套“粗网格 + 细网格”的双网格方案，解决当前覆盖面积统计因重复积分而高估、进而导致覆盖率超过 100% 与漏测率为负的问题。

本方案同时要求：

1. 保留现有粗网格在覆盖次数计算与分区阶段中的作用
2. 新增分区级细网格，用于测线规划过程中的覆盖状态更新
3. 将最终统计口径改为基于细网格覆盖状态的去重面积统计

---

## 2. Deliverable (交付物)

### 2.1 架构交付物

- 一套双网格架构设计：
  - 粗网格：用于 `Coverage.py` 与 `Partition.py`
  - 细网格：用于分区内部的覆盖状态追踪与统计

### 2.2 算法交付物

- 分区细网格生成规则
- 细网格状态更新规则
- 分区统计与全局统计口径
- `step` 与 `microstep` 的新参数关系设计

### 2.3 未来实现交付物

- 新增细网格状态模块
- 改造 `Planner.py` 的分区规划状态更新流程
- 改造指标统计模块，替换当前积分口径

---

## 3. Constraints (约束条件)

### 3.1 已确认的设计选择（来自用户）

1. **细网格用途**：用于 **统计 + 规划过程中的覆盖状态更新**（不要求首版直接接入终止条件）
2. **细网格边长基准**：`l = c_l * w_min_partition`，其中 `c_l >= 1`
3. **cell 覆盖判定**：采用 **采样判定**，而非仅中心点或精确几何并集
4. **部分覆盖面积口径**：按 **采样覆盖比例** 计入面积
5. **规划步长**：采用 `step = c_step * l`
6. **坡度差分步长**：采用 `microstep = clip(l / 2, microstep_min, microstep_max)`
7. **统计口径**：分区内统计 + 全局再次汇总统计

### 3.2 保持兼容的范围

- 不推翻当前 `GridCell -> Coverage -> Partition -> Planner` 的四阶段主线
- 不要求首版细网格直接替换粗网格的覆盖次数矩阵
- 不要求首版细网格直接驱动终止条件

### 3.3 已冻结的参数标定（来自用户）

以下参数已由用户显式选择并冻结为实现默认值：

- `c_l = 1.5`
- `max_cells_per_partition = 100000`
- `c_step = 1.0`
- `cell_sampling_mode = 9-point`
- `partial_threshold_full = 0.9`
- `partial_threshold_any = 0+`（即只要覆盖比例大于 0 就算 partial）
- `microstep_min = 10`
- `microstep_max = 70`
- `w_min_partition` 口径：**在该分区粗网格中找到深度最小点，并取该点的覆盖宽度**
- `metrics_migration_mode = dual_output`（旧积分口径与新细网格口径并行输出）

### 3.4 参数实现约束

即便默认值已冻结，下列参数仍应以可配置字段或构造参数方式实现，而不是散落在逻辑分支中：

- `c_l`
- `max_cells_per_partition`
- `c_step`
- `cell_sampling_mode`
- `partial_threshold_full`
- `microstep_min`
- `microstep_max`

## 4. Acceptance Criteria (验收标准)

### 4.1 双网格边界清晰
- [ ] 粗网格继续用于覆盖次数计算与分区
- [ ] 细网格仅在分区规划阶段生成与更新
- [ ] 粗网格和细网格职责不混淆

### 4.2 细网格构建规则明确
- [ ] 每个分区可独立生成一张细网格
- [ ] `w_min_partition` 的计算口径明确
- [ ] `l = c_l * w_min_partition` 明确且参数化
- [ ] 对异常过细情况有保护策略（如最小允许边长或预警）

### 4.3 规划状态更新明确
- [ ] 每新增一个测线段，就能更新相关细网格覆盖状态
- [ ] cell 状态至少能区分：未覆盖 / 部分覆盖 / 充分覆盖
- [ ] 覆盖状态支持重复覆盖去重

### 4.4 统计口径正确
- [ ] 分区覆盖面积由细网格状态推导，而不是测线宽度积分累加
- [ ] 全局覆盖面积不超过待测海域总面积
- [ ] 覆盖率被约束在 `[0, 1]`
- [ ] 漏测率被约束在 `[0, 1]`
- [ ] 不再出现负数漏测率

### 4.5 参数联动明确
- [ ] `step = c_step * l` 的实现位置明确
- [ ] `microstep = clip(l/2, min, max)` 的实现位置明确
- [ ] 细网格、规划步长、坡度差分三者关系可解释

## 5. Product Acceptance Criteria (产品验收标准)

- 统计结果应从“积分代理值”切换到“去重后的离散覆盖面积”
- 指标输出应对高重叠率和测线相交具有稳定性
- 细网格状态能够为未来补线策略或终止条件升级留下扩展接口

---

## 6. Manual Spot Checks (手动验证点)

1. 构造相交测线案例，确认覆盖率不超过 100%
2. 构造高重叠平行测线案例，确认覆盖面积不随重复次数线性膨胀
3. 检查分区统计与全局统计是否一致（全局总面积不大于总海域面积）
4. 对比同一分区在不同 `c_l` / `c_step` 下的统计稳定性
